Where Patient Data Gets Duplicated

The central issue is not just multiple screens. It is that patient identity is copied into several stores instead of referenced from one canonical record.

Canonical VSP Patient

Best current source of truth. Downstream flows reuse it through pickers, but most of them copy fields forward into local case or draft data instead of keeping a single patient reference.

id / mrn
demographics
history / meds / allergies

Unified Target Funnel

This is the simplest streamlined experience that still preserves profession-specific note forms and shared outputs.

Step 1 Select patient

Choose an existing patient or create one patient record once.

Step 2 Keep one patient ID

Carry a single canonical patientId through every downstream flow.

Step 3 Select note type

Pick PT Evaluation, PT SOAP, Dietetics NCP, or future note templates.

Step 4 Create note envelope

{ patientId, encounterId, professionId, templateId, status }

Step 5 Open shared shell

Use one workspace shell with profession-specific content inside it.

Step 6 Use shared outputs

Save, sign, export, amend, and Case File all operate on the same note contract.

Step 7 Show chart timeline

Patient workspace index becomes a true cross-discipline timeline, not an aggregator of copies.

Top Unification Moves

Make patient creation singular Every funnel should begin with select/create patient, not by synthesizing local copies in each discipline.
Adopt one note storage contract Replace split keys like draft_<caseId>_<encounter> and dietetics_draft_<caseId> with one envelope abstraction.
Separate patient, encounter, and note concerns Setting, acuity, diet order, and case context should attach to encounter or workflow context instead of becoming alternate patient stores.
Stop hidden PT patient synthesis The multidisciplinary shell should create a PT encounter for the same patient, not a new blank PT patient id.
